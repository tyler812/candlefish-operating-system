Executive Summary

Candlefish operates as a dual-engine organization: Patrick drives Spark & Elevation while our System delivers Integration & Finish. This charter codifies how we ship remarkable products predictably without sacrificing creative edge.

Stage-Gate Framework

Progression: Spark → Seed → Scaffold → Ship → Scale

Spark Gate

Capture and qualify ideas

Entry: Problem + Hypothesis Exit: Strategic Fit ≥3/5 Auto-kill: 30 days inactive

Seed Gate

Validate feasibility

Entry: Approved Spark Exit: Technical spike complete Auto-kill: 14 days blocked

Scaffold Gate

Build foundation

Entry: Approved Seed Exit: Core working, tests >60% Auto-kill: 7 days failing

Ship Gate

Polish and release

Entry: Scaffold complete Exit: Production deployed Auto-kill: 48hr customer bugs

Scale Gate

Optimize and expand

Entry: 30-day stability Exit: Performance targets met Auto-kill: 3-day SLA violations

WIP Limits

  • Per Pod: Maximum 3 items in Scaffold
  • Per Person: Maximum 2 items owned
  • Cross-Pod: Maximum 5 Seed explorations
  • Portfolio: Maximum 10 total in flight

Operating Cadences

  • Daily: Unblock & Commit (10:00 PST, 15 min)
  • Weekly: Demo Friday (Fridays 13:00 PST)
  • Monthly: Portfolio Council (First Monday 11:00 PST)

Minimum Remarkable Standard

Every shipped feature must be:

  1. Story Clear: User can explain value in one sentence
  2. Performance Floor: <100ms interaction, <3s load
  3. Accessibility: WCAG 2.1 AA compliant
  4. Observable: Metrics dashboard exists
  5. Supported: Runbook and error states documented
  6. Delightful: One moment of unexpected polish
